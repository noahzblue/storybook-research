## Story Analyses

### Grid (default)

- **Intent:** Baseline — shows Grid with real ContentItem children at default breakpoint configuration.
- **Canvas:** Grid of "Reported Missing: Series 2" content cards — multiple ContentItem components as children. Rich visual output via children, not the Grid itself.
- **Controls (4 knobs):**
  - Number of items: `8` (number input)
  - className: `"gel-wrap"` (textarea)
  - itemsPerRow: `{"xs": 2, "m": 3, "xl": 4}` (textarea/JSON)
  - fullRows: `false` (checkbox)
- **What's hidden:** children (generated by story), minVisible, setRef
- **KEY FINDING:** "Number of items" is NOT a component prop — it's a story-level helper that controls how many ContentItem children the story generates. BBC adds story-level knobs to control children count without exposing `children` as a raw prop knob.
- **KEY FINDING:** Layout component canvas = rich because of CHILDREN, not the layout itself. Grid renders as invisible structure. All visible content comes from child ContentItems.
- **Actions:** None.
- **Accessibility:** 0 violations.
- **Interactions:** None.

---

### Complete Rows

- **Intent:** Show `fullRows=true` behaviour — Grid hides orphan items to maintain complete rows only.
- **Canvas:** Same ContentItem grid, but with an odd starting count (11 items) — some cards hidden to fill only complete rows.
- **Controls (4 knobs — same set, different values):**
  - Number of items: `11` (number input — intentionally odd/incomplete at some breakpoints)
  - className: `"gel-wrap"` (textarea)
  - itemsPerRow: `{"xs": 2, "m": 3, "xl": 4}` (textarea/JSON)
  - fullRows: `true` (checkbox — the discriminating prop)
- **Per-variant note:** "Show full rows based on items per row." Used: Search page (with min number), Category Highlights.
- **KEY FINDING:** 11 items chosen deliberately — with 4-per-row at xl, 11 doesn't divide evenly. The odd count makes `fullRows` meaningful: with `fullRows=true` the incomplete last row is hidden. Without it you'd see a ragged partial row.
- **Actions:** None.

## Theming / Tokens

- **Grid itself is invisible.** No background, no border, no visual marker. Canvas content = children only.
- **className `"gel-wrap"`:** GEL wrapper class — adds max-width and horizontal padding to contain the grid. Not a component prop default — set explicitly in stories.
- **Breakpoints via `itemsPerRow`:** Object keys = named breakpoints (xs, m, xl). Same breakpoint system as ContentItem's required `itemsPerRow` prop — BBC shares the same object shape across components.

## Patterns & Observations

1. **Layout components answer "what problem does fullRows solve?"** Grid has 2 stories — not 2 visual states, but 2 grid modes: default (ragged last row ok) vs fullRows (clean complete rows). The story teaches when you'd want each.

2. **Story-level "Number of items" knob = children generator.** BBC adds a non-component-prop knob to control story complexity (how many children render). This is a story composition technique — the knob is in the story, not the component. You change item count to stress-test the layout, not to configure the Grid component.

3. **Odd item count is intentional.** Complete Rows uses 11 items (not 12, not 10) because 11 ÷ 4 = 2 remainder 3. The fractional last row is the POINT — `fullRows` hides it. 12 items would show identical output whether `fullRows` is true or false.

4. **Rich canvas via children, not the layout itself.** Grid stories look visually rich (real content cards). But the layout component contributes nothing visual. The appearance comes from children (ContentItem). Layout = invisible structural wrapper.

5. **Minimal story count for structural components.** 2 stories for Grid — the fewest of any component studied. Structural components have fewer visual states to demonstrate. The only meaningful variant is the one feature (`fullRows`) that changes visible output.

6. **Related components: N/A.** Grid is the only component with no related components listed. It's a pure structural primitive — not composed with other named components in the docs.

7. **`itemsPerRow` as JSON textarea.** Multi-breakpoint config object exposed as raw JSON in a textarea knob. BBC doesn't use multiple separate knobs for xs/m/xl — one JSON object for the whole responsive config.

8. **`minVisible` hidden from all stories.** The companion prop to `fullRows` never appears in knobs. Likely considered too advanced/optional — the basic `fullRows=true` story demonstrates the concept without it.

9. **Largest "current usage" list.** Grid appears on more pages than any other component (10 page types listed). Makes sense — every content-grid page uses it. Broad utility = sparse story count.

## Key Findings Summary

- **Story count:** 2 stories (fewest of all components studied)
- **How layout behaviour is demonstrated:** Rich children (ContentItems) fill the canvas. The grid structure is invisible — you see the layout's EFFECT via child positioning.
- **Responsive grid stories:** Not explicit viewport-change stories. `itemsPerRow` JSON knob + resize the canvas manually to see breakpoint behaviour.
- **Children generation:** Story-level "Number of items" knob generates N children programmatically. Not a component prop — a story composition technique.
- **fullRows story:** Odd item count (11) chosen intentionally to demonstrate incomplete-row hiding.
- **0 required props:** Fully optional API — but meaningless without `itemsPerRow` and children.
- **Docs style:** Same prescriptive structure. "See variant for specific use case." Related components: N/A (unique).
